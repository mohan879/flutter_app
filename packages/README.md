# Packages

This directory contains a collection of Dart/Flutter analyzer plugin packages
and example configurations. These packages provide lint rules, code generation
examples, and best practices for maintaining code quality and architecture in
Flutter projects.

## Available Packages

- [**avoid_banned_exports**](avoid_banned_exports/README.md) - Rules and
  patterns for controlling and preventing banned exports in your codebase
- [**avoid_banned_file_names**](avoid_banned_file_names/README.md) - Enforces
  naming conventions and prevents use of banned or generic file names
- [**avoid_banned_imports**](avoid_banned_imports/README.md) - Controls import
  patterns and prevents importing from restricted paths and modules
- [**avoid_banned_types**](avoid_banned_types/README.md) - Prevents usage of
  banned types and enforces type safety boundaries
- [**avoid_deprecated_usage**](avoid_deprecated_usage/README.md) - Detects and
  helps migrate away from deprecated APIs and libraries
- [**banned_usage**](banned_usage/README.md) - General framework for banning
  dangerous APIs and enforcing custom wrappers
- [**example_analyzer_plugin**](example_analyzer_plugin/README.md) - Example
  implementation of a custom Dart analyzer plugin
- [**max_imports**](max_imports/README.md) - Monitors and limits the number of
  imports per file to manage complexity
- [**metrics_examples**](metrics_examples/README.md) - DCM metrics configuration
  examples demonstrating advanced threshold management and team workflows

## Overview

These packages are designed to work with the Flutter test cases framework and
demonstrate various lint rule implementations, architectural patterns, and code
quality enforcement strategies for large-scale Flutter applications.
